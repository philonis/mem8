如果记忆足够重要，它就不该默认放在别人的硬盘上。

这不是一句口号，而是 mem8 的产品前提：记忆需要 persistence，也需要 ownership。

核心价值，不只是“记忆”

把 mem8 最硬的差异化讲透：本地优先、零额外运维、还能直接省 Token。

01

本地持久记忆

用 SQLite 做默认后端，跨会话、跨重启保留记忆，不依赖任何云数据库或远程服务。

02

Token 节省可量化

不是“多一个记忆层”而已。通过结构化召回和记忆压缩，离线 benchmark 已看到约 36% 的 Token 降幅。

03

零额外运维

没有向量数据库、没有同步脚本、没有远程控制面板。安装完成就能写入、召回和清理记忆。

04

自动 hygiene

内置相似记忆合并、陈旧噪音清理、compact 优先保留高价值记忆，避免“越用越脏”。

为什么这组数据重要？

很多“记忆系统”只能证明自己记住了，却不能证明自己让系统更高效。mem8 的价值在于：它不只是补齐记忆，更是在降低长期 prompt 成本。

  • 长会话更稳，少遗忘
  • 上下文更轻，少浪费
  • 更适合本地、隐私敏感、长期使用场景

安装部分尽量简单到“不用解释”

首屏只保留一个安装入口；下面再给补充说明，避免按钮太多把注意力打散。

1

把这个链接发给 OpenClaw

这是最适合作为安装入口的地址：直接返回 SKILL.md 原文，便于 OpenClaw 读取和执行安装流程。

https://raw.githubusercontent.com/philonis/mem8/main/SKILL.md
2

安装后先决定是否导入旧记忆

如果你已经在 OpenClaw 里积累了 `MEMORY.md` 或 `workspace/memory/*.md`，建议安装完成后显式选择是否迁移；不要默认自动导入。

node scripts/mem8-cli.js import-openclaw \ --db ~/.openclaw/memory/mem8.db \ --memoryMd ~/.openclaw/workspace/MEMORY.md \ --memoryDir ~/.openclaw/workspace/memory
3

如果你要手动安装

只有在特殊网络环境或要手动控制目录时，才需要下面这组命令。

git clone https://github.com/philonis/mem8.git ~/.openclaw/plugins/mem8 cd ~/.openclaw/plugins/mem8 && npm install openclaw restart
4

默认配置即可启动

默认使用 SQLite 和 Ollama，本地目录不存在时会自动创建；如果你要更严格控制组装预算，可以直接设置 `maxTokensPerAssemble`。

{ "embeddingProvider": "ollama", "embeddingModel": "nomic-embed-text:latest", "embeddingUrl": "http://127.0.0.1:11434", "dbPath": "~/.mem8/memories.sqlite", "maxTokensPerAssemble": 500, "debug": false }
