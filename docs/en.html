Memory infrastructure with a sharper point

mem8 is not only about persistence. It is also about efficiency, privacy, and reducing prompt waste in real OpenClaw sessions.

01

Local persistent memory

SQLite is the default backend, so memory survives restarts and sessions without any external service.

02

Measured token savings

This is not just “one more memory layer.” Structured recall and compaction already show measurable token reduction in offline benchmarks.

03

No extra ops

No vector database, no sync scripts, no remote control plane. Install it and start writing, recalling, and cleaning memory.

04

Automatic hygiene

Similar memories merge, stale noise gets pruned, and compaction favors high-value records so the store does not decay over time.

Install and get started

Use the steps below to install mem8, migrate legacy memory when needed, and apply the default config.

1

Send this URL to OpenClaw

Use this link to load `SKILL.md` directly and start plugin installation.

https://raw.githubusercontent.com/philonis/mem8/main/SKILL.md
2

Decide whether to import legacy memory

If you already have `MEMORY.md` or `workspace/memory/*.md` data in OpenClaw, choose explicitly whether to migrate it after installation instead of importing automatically.

node scripts/mem8-cli.js import-openclaw \ --db ~/.mem8/memories.sqlite \ --memoryMd ~/.openclaw/workspace/MEMORY.md \ --memoryDir ~/.openclaw/workspace/memory
3

Manual install if needed

Only needed for special network environments or when you want direct control over plugin placement.

Directory model: plugin code lives in ~/.openclaw/plugins/mem8, while runtime data lives in ~/.mem8/ (default DB: ~/.mem8/memories.sqlite).

git clone https://github.com/philonis/mem8.git ~/.openclaw/plugins/mem8 cd ~/.openclaw/plugins/mem8 && npm install openclaw restart
4

Default config is enough

SQLite and Ollama are the defaults. If you want a stricter assembly budget, set `maxTokensPerAssemble` directly.

{ "embeddingProvider": "ollama", "embeddingModel": "nomic-embed-text:latest", "embeddingUrl": "http://127.0.0.1:11434", "dbPath": "~/.mem8/memories.sqlite", "maxTokensPerAssemble": 500, "debug": false }

mem8 uninstall guide

If you need to roll back or troubleshoot quickly, use these commands directly.

1

Standard uninstall

# Preview (optional) openclaw plugins uninstall mem8 --dry-run # Uninstall openclaw plugins uninstall mem8 --force
2

Emergency uninstall

If OpenClaw fails to start because plugin loading is stuck on an error, run in order:

# 1. Remove the plugin directory rm -rf ~/.openclaw/plugins/mem8 # 2. Remove plugin reference from config openclaw config unset plugins.entries.mem8 # 3. Restart Gateway openclaw gateway restart
3

Clean up data (optional)

mem8 data and indexes are stored in ~/.mem8/ by default. If you no longer need them after uninstall:

rm -rf ~/.mem8
4

Restore original memory system

After mem8 is removed, OpenClaw automatically goes back to the original MEMORY.md + memory/*.md file-based memory flow. No extra action is required.
